# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.0.2b3 (current) | :white_check_mark: |
| 0.0.2b2 | :white_check_mark: |
| 0.0.2b1 | :white_check_mark: |
| 0.0.1b2 | :x: (upgrade recommended) |
| 0.0.1-alpha | :x: (upgrade recommended) |
| 0.x (future) | :white_check_mark: |
| 1.0+ (future) | :white_check_mark: |

Pre-release (0.x) is supported on a best-effort basis; no long-term support guarantee.

## Reporting a Vulnerability

**Do not report security vulnerabilities in public GitHub Issues.**

Report them **privately** only:

- **GitHub:** Use [Private vulnerability reporting](https://github.com/R3zeProjects/vapi_osp/security/advisories/new) for this repository (if enabled), or  
- **Email:** Send to the address listed in [CONTACT.txt](CONTACT.txt) (Official Channel). Use the subject line **Security report – VAPI OSP**.

Include: description and impact, steps to reproduce, affected version(s), and any suggested fix.

- **Initial response:** within 7 business days.  
- **If accepted:** we will work with you on a fix and coordinated disclosure (e.g. advisory).  
- **If declined:** we will explain why; you may disclose after the response if you wish.
