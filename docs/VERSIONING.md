# Versioning scheme

The project uses the following version numbering:

| Version part | Meaning | Example |
|--------------|---------|---------|
| **0.** (first digit after major) | Released version | 0.1.0, 0.2.0 |
| **0.0** | Beta | 0.0.1, 0.0.1b1 |
| **0.0.1** | Alpha | 0.0.1 (alpha) |
| **b** (suffix) | Patch / fix | b1, b2 → 0.0.1b1, 0.0.1b2 |

**Current API version:** 0.0.1-alpha (first alpha).

Sequence: **0.0.1-alpha** (current, first alpha) → **0.0.1b1** (patch) → **0.1.0** (first stable).
