# Процесс релиза

- **Версия:** задаётся в `CMakeLists.txt` (`project(vapi_osp VERSION x.y.z)`). Обновляйте при релизе. **Текущая версия API:** 0.0.2b3. Схема: [VERSIONING.md](VERSIONING.md).
- **До 1.0.0:** возможны несовместимые изменения без смены мажорной версии.
- **С 1.0.0:** соблюдаем [Semantic Versioning](https://semver.org/): мажор — несовместимые изменения API, минор — новая функциональность с сохранением совместимости, патч — исправления.

## Шаги релиза

1. Обновить версию в `CMakeLists.txt`.
2. Собрать и прогнать тесты: `cmake -B build && cmake --build build && ctest --test-dir build`.
3. Создать тег (например `v0.0.1-alpha`) и пуш, либо создать Release в репозитории.
4. При распространении через пакетные менеджеры обновить их конфигурацию (vcpkg, Conan и т.д.), если используется.

## Поддерживаемые конфигурации

Официальная платформа распространения — **Windows** (MSVC, MinGW). Бинарные файлы публикуются для Windows. Подробнее — [user_guide.md](user_guide.md).
