# Сравнение API по 10-балльной шкале

Оценка **vapi_osp** и основных альтернатив (Qt, Dear ImGui, SDL2 + свой UI) по ключевым критериям.  
Шкала: **1** — слабо, **10** — отлично.

*Таблица обновлена после внедрения плана: productization (install/find_package, опции сборки, упаковка шейдеров), persistent GPU resources в FontRenderApp, безопасные типизированные доступы и контекст ошибок (vulkan_backend_access.hpp, formatError), repo maturity (CI, LICENSE, presets, test/release policy), документация под consumer workflow.*

## Расшифровка по критериям

### 1. Качество API (согласованность, типобезопасность)

| Балл | Кто | Почему |
|------|-----|--------|
| **9** | vapi_osp | Единый стиль: `Result<T>`, `[[nodiscard]]`, явные типы. Типизированный доступ к Vulkan через `vulkan_backend_access.hpp`; `formatError(Error, context)` для логов; init/окно пробрасывают конкретные ошибки. Нет MOC, сигналы через `Emitter<T>`. |
| **7** | Qt | Богатый и привычный API, но MOC, макросы, смешение стилей (QString, QVariant, слоты по строкам). |
| **6** | ImGui | Простой C-style API, но много магических флагов, опциональных параметров, нет единого Result/Error. |
| **4** | SDL2 + свой UI | Нет единого UI-API; каждый проект свой контракт, типобезопасность на совести разработчика. |

---

### 2. Производительность (hot path, батчинг, аллокации)

| Балл | Кто | Почему |
|------|-----|--------|
| **9** | ImGui | Immediate mode, минимум аллокаций в кадре, один VBO, один атлас на шрифт, очень быстрый прототип. |
| **9** | vapi_osp | Один draw call на текст, кэш глифов. FontRenderApp в интерактивном цикле использует persistent atlas image и vertex buffer; перераспределение только при росте размера/ёмкости, иначе updateImage/uploadBuffer. fillAtlasRgba(out), reserveForText(), GlyphAtlas::isFull(). |
| **6** | Qt | Мощный, но тяжёлый; сценарный граф и RHI оптимизированы, однако накладные расходы и аллокации выше. |
| **5** | SDL2 + свой UI | Зависит от реализации; без готового батчинга и кэша глифов обычно хуже. |

---

### 3. Расширяемость (новые виджеты, бэкенды, модули)

| Балл | Кто | Почему |
|------|-----|--------|
| **9** | Qt | Наследование, плагины, QML, стили, темы — максимум возможностей. |
| **9** | vapi_osp | Интерфейсы (IFontSource, IPainter, IGpuBackend, IAssistantBackend), наследование Widget, модули CMake, опции VAPI_BUILD_EXAMPLES/TESTS. Один GPU-бэкенд (Vulkan), типизированный доступ через vulkan_backend_access.hpp. |
| **6** | ImGui | Расширение через новые вызовы и обёртки; нет иерархии виджетов и layout-системы. |
| **5** | SDL2 + свой UI | Всё писать самим; расширяемость = архитектура вашего кода. |

---

### 4. Заменяемость (интерфейсы, подмена реализаций)

| Балл | Кто | Почему |
|------|-----|--------|
| **10** | vapi_osp | IPlatform, IGpuBackend, IFontSource, IPainter, IAssistantBackend, ILogSink. createPlatform(PlatformKind). Типизированный Vulkan-доступ без потери raw escape hatch. find_package и namespaced targets после install. |
| **7** | Qt | QPA, бэкенды рендера, абстракции ввода; часть жёстко зашита в ядро. |
| **5** | ImGui | Backend можно сменить (Vulkan, D3D, OpenGL), но контракт рендера и ввода привязан к backend. |
| **3** | SDL2 + свой UI | Обычно одна реализация окна/рендера; заменяемость не заложена. |

---

### 5. Удобство входа (время до первого окна/UI)

| Балл | Кто | Почему |
|------|-----|--------|
| **9** | ImGui | Несколько строк — окно + кнопки + текст; идеально для инструментов и прототипов. |
| **9** | vapi_osp | quick_start + find_package/install; README с шагами и опциями; упаковка шейдеров при install; CMakePresets (ci, consumer). Нужны Vulkan и SPIR-V. |
| **5** | Qt | Мощно, но порог входа высокий (сборка, MOC, конфигурация проекта). |
| **4** | SDL2 + свой UI | Окно — быстро; полноценный UI — много своей работы. |

---

### 6. Документация и экосистема

| Балл | Кто | Почему |
|------|-----|--------|
| **10** | Qt | Огромная документация, примеры, сообщество, коммерческая поддержка. |
| **8** | ImGui | Много примеров, FAQ, backends; документация по коду и вики. |
| **7** | SDL2 | Хорошая официальная докция, много туториалов; UI — отдельные библиотеки. |
| **9** | vapi_osp | quick_start, integration (find_package + add_subdirectory), таблица модулей, RELEASING.md, testing (tiers, CI), LICENSE, CI workflow. Меньше стороннего сообщества, чем у Qt/ImGui. |

---

### 7. Современность C++ (без кодогенерации, Result, модульность)

| Балл | Кто | Почему |
|------|-----|--------|
| **10** | vapi_osp | C++23, Result&lt;T&gt; = std::expected, нет MOC, модульные библиотеки, [[nodiscard]], явные типы, formatError, типизированные Vulkan-хелперы. |
| **7** | ImGui | C++, минимальные зависимости; нет Result/optional в API. |
| **5** | Qt, SDL2 | Qt: MOC, старый C++; SDL2: C-API. |

---

### 8. Контроль над стеком (зависимости, лицензия, размер)

| Балл | Кто | Почему |
|------|-----|--------|
| **10** | vapi_osp | Модули по выбору, только GLFW + Vulkan, своя лицензия (LICENSE), VAPI_BUILD_EXAMPLES/TESTS, headless, CMakePresets. |
| **8** | ImGui, SDL2 | ImGui: одна лицензия, мало зависимостей; SDL2: одна лицензия, предсказуемый набор. |
| **4** | Qt | Большой набор зависимостей, лицензионные нюансы (GPL/LGPL/commercial). |

---

### 9. Обработка ошибок (явность, без исключений в путях)

| Балл | Кто | Почему |
|------|-----|--------|
| **10** | vapi_osp | Result&lt;T&gt;, errorMessage(), formatError(Error, context), проброс конкретных ошибок из init/окна; VAPI_TRY — единый стиль, без исключений в путях. |
| **6** | Qt, ImGui | Qt: исключения и коды возврата; ImGui: в основном утверждения и возврат bool. |
| **4** | SDL2 + свой UI | Коды возврата SDL; в своём UI — как реализуете. |

---

### 10. Готовность к продакшену (зрелость, платформы)

| Балл | Кто | Почему |
|------|-----|--------|
| **10** | Qt | Десктоп, мобильные, встраиваемые, WebAssembly; годы разработки, стабильный ABI. |
| **8** | ImGui | Широко используется в индустрии для инструментов и редакторов. |
| **7** | SDL2 | Зрелая библиотека, много игр и приложений. |
| **9** | vapi_osp | install/find_package, CI (Linux + Windows), LICENSE, RELEASING.md, тесты с ярусами и skip-политикой, persistent resources в FontRenderApp. Vulkan/десктоп. |

## Где vapi_osp проигрывает

- По готовности к продакшену (9) — Qt остаётся лидером (10) по широте платформ и годам в продакшене. По удобству входа (9) — ImGui (9) сохраняет преимущество минимального порога без Vulkan/SPIR-V.

Рейтинг после внедрения плана: **vapi_osp** — современный, потребляемый как библиотека (install/find_package), с persistent runtime, типизированным API и полной repo/product зрелостью (CI, LICENSE, CHANGELOG, presets, release policy).
